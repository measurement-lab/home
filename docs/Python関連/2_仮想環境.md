## 1_仮想環境が必要となる背景

- Python環境を構築するには，Miniforgeリポジトリを使い，パッケージのインストールはcondaコマンドを使うことは既に説明しました．
- condaコマンドをネットで調べてみると，必ず，**仮想環境**という用語が目についてきます．仮想環境とは何なんでしょうか．どのようなケースで必要となってくるのでしょうか．ここでは，仮想環境について，なぜ必要なのかをまず説明します．

### 1.1_パッケージのバージョンの通常の指定方法

- 例えば，visautilsパッケージは単独のPythonスクリプトではなく，numpyパッケージ，matplotlibパッケージ他，幾つかの別のパッケージが必要（内部で使っている）です．
- これらの別パッケージの使っている機能は，visautilsパッケージを実装（作成）した段階での，それらの別パッケージのバージョンで決めます．つまり，visautilsパッケージを実装した段階で，numpyパッケージのバージョンが2.4だったなら，2.4未満のnumpyパッケージでは上手く動作しない（機能が不足している）かもしれません．
- そこで，visautilsパッケージを始めとして，Pythonのパッケージは，インストールする際に，必要となる別パッケージと，そのバージョンの条件を確認する機能があります．上記の例であれば，「**バージョン2.4以降のnumpyパッケージを必要**とする」なんて具合です．
- 必要となる別パッケージがインストールされていない場合は，バージョンも満足する状態で別パッケージを**自動で追加インストール**してくれます．

### 1.2_特定のバージョンを指定するパッケージ

- Pythonのパッケージは普通，**後方互換性**を満足するように作られています．つまり，建前上は，**新しいバージョンのパッケージは，古いバージョンのパッケージを置き換える**ことができます．
- すると，**全てのパッケージが最新のバージョンになるようにしておけば良い**（問題無い）ことになるのですが，実際には，幾つかのパッケージで，後方互換性を満足しないケースが存在します．つまり，「○○○のパッケージは，バージョンx.y.zに限定する」なんてことが実際にはあります．
- 例えば，このドキュメントを作成するのに使っている，mkdocs-to-pdfパッケージは，バージョン62.3のweasyprintパッケージのみ必要としています．実際に，mkdocs-to-pdfパッケージをインストールすると，バージョン62.3のweasyprintパッケージがインストールされます．しかし，2026年4月現在，最新のweasyprintパッケージのバージョンは68.1です（condaコマンドでインストールできる最新バージョンは67.0です）．

### 1.3_パッケージ間で異なるバージョンが必要となるケース

- 上記に紹介したmkdocs-to-pdfパッケージのケースだと，mkdocs-to-pdfパッケージをインストールして，weasyprintパッケージがインストールされていなければ，バージョン62.3のweasyprintパッケージが自動で追加インストールされます．
- しかし，weasyprintパッケージを必要とする別のパッケージがあり，それがバージョン63以降のweasyprintパッケージを必要とした場合，どうなるのでしょうか．実際に，そのような状況になった場合，どのようになるのは良く分かりません．パッケージのインストールに失敗するのか（これが一番可能性が高いです），インストールには成功するが，いずれかのパッケージを実行した際に不具合が発生してしまうのか（Pythonの仕様として決まっているのかもしれませんが，そこまで詳しく知りません）． 

## 2_独立したPython環境を提供する仮想環境

- 上記に説明したように，Pythonには便利なパッケージが大量に提供されている反面，それらのパッケージは，別のパッケージを必要とし，それらの必要としているパッケージのバージョンが競合（不一致）となるケースがあります．
- これを回避するのが，仮想環境という仕組みです．仮想環境というのは，**名前を付けてPython環境を丸ごと新たに構築する仕組み**です．パッケージのバージョンが競合するようなケースでは，新たに仮想環境（別のPython環境）を構築して，そちら側でパッケージをインストールします．
- 仮想環境の構築や，切り替えなどは全てcondaコマンドを使って行うことができます．

### 2.1_condaコマンドを使った仮想環境の操作

- 下記にcondaコマンドを使った仮想環境の操作手順を示します．

| 目的           | コマンド                                                                                    | 説明                                                                                |
| -------------- | ------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 仮想環境の一覧 | **conda info -e**                                                                           |                                                                                     |
| 仮想環境の追加 | **conda create -n 仮想環境名 python=x.xx**                                                  | 仮想環境名にコンマは使えない<br>（**Python3.9**などコンマを使った仮想環境名はダメ） |
| 仮想環境の変更 | **conda activate 仮想環境名**（仮想環境へ移動）<br>**conda deactivate**（仮想環境から戻る） |                                                                                     |
| 仮想環境の削除 | **conda remove --name 仮想環境名 --all**                                                    |                                                                                     |

### 2.2_仮想環境の構築の手順例

- 具体的に，condaコマンドを使って仮想環境を構築してみます（構築した仮想環境は簡単に削除できますので，実際にやってみても問題ありません）．
- まず最初に，Miniforge Promptで，下記のコマンドを実行してみて下さい．

```txt
conda info -e
```

- おそらく，**base \* c:\users\ユーザ名\miniforge3** といった具合に表示されたと思います．これは，まだ何も仮想環境が無い状態を示しています．**base**というのは仮想環境ではなくベースであるPython環境で，その次の **\*** は，ベース環境がアクティブ（有効）であることを意味し，それ以降は，そのPython環境のインストール先のパスです．

```txt
base * c:\users\ユーザ名\miniforge3
```
- では，新たに別の仮想環境を追加してみましょう．ここでは簡単のため，仮想環境名をtestとします．上記の説明にも記述してあるように，新たに追加する仮想環境では，ベース環境とは別のバージョンのPythonをインストールすることも可能です（同じバージョンでも問題ありません）．但し注意すべき点は，仮想環境名にコンマは使えません．Pythonのバージョンを変えた仮想環境を追加しようとして，Python3.xxとかいう仮想環境名を使ってしまわないように注意しましょう（経験則）．

```txt
conda create -n test python=3.13
```
- 続けて，下記のコマンドを再実行して，作成した仮想環境が正しく追加されているかどうか確認してみて下さい．
- 先ほど追加で構築した仮想環境が，今度は表示されるはずです．

```txt
conda info -e

base * c:\users\ユーザ名\miniforge3
test   c:\users\ユーザ名\miniforge3\envs\test
```

- 仮想環境が正しく追加されていることを確認したら，今度は，追加した仮想環境に切り替えてみましょう．

```txt
conda activate test
```

- 仮想環境へ問題なく切り替えられたら，プロンプト名がbaseから仮想環境名に変更になっているハズです．さらに，下記のコマンドを実行してみて下さい．

```txt
conda info -e

base   c:\users\ユーザ名\miniforge3
test * c:\users\ユーザ名\miniforge3\envs\test
```

- 今度は，仮想環境の方に\*が付いているハズです．これは，仮想環境側がアクティブ（有効）となっていることを表しています．
- 仮想環境に切り替えたら，ベース環境と同じく，必要となるパッケージを一通りインストールします．残念ながら，ベース環境でインストールしたパッケージ群は，仮想環境側ではインストールされていません．ベース環境をインストールした直後の状態と同じく，追加した直後の仮想環境は，何もパッケージがインストールされていない状態なのです．

- バージョンが競合するようなパッケージは，それ専用の仮想環境を構築し，その仮想環境に切り替えた後，該当するパッケージをインストールするようにすれば，バージョンの競合を回避することができます．

### 2.3_仮想環境からベースに戻る手順

- 仮想環境からベースに戻るのも，condaコマンドを使います．下記のコマンドを実行して下さい．

```txt
conda deactivate
```

- また，Miniforge Promptを閉じても，仮想環境からベースに戻ります．これは，逆にいうと，Miniforge Promptを開く度に，仮想環境に切り替える必要があるということです．

### 2.4_仮想環境の削除

- 作成した仮想環境は，condaコマンドで簡単に削除できます．
- まず，ベースに切り替えた状態で，下記のコマンドを実行して下さい．該当する仮想環境が（インストールしたパッケージも含めて全て）削除されます．

```txt
conda remove --name 仮想環境名 --all
```
