## 1_condaコマンドとpipコマンドの違い

- Pythonは，数多くの有益なパッケージを利用できるのが最大の特長となっています．ところが，これらのパッケージはPythonの開発元とは別の，第三者によって開発および提供（公開）されています．
- これらのパッケージをインストールするには，主に，**conda**コマンドと**pip**コマンドという2つのコマンドが使われます（他にもあります）．これらのコマンドの違いは何なんでしょうか．
- 最大の違いは，パッケージのリポジトリ（インストールするパッケージが収納先されている場所）が異なる点です．condaコマンドでインストールするパッケージはconda-forgeリポジトリ，pipコマンドでインストールするパッケージはPyPIリポジトリとなっています．但し，**visautils**パッケージのように，ローカルファイルからインストールするケースでは，pipコマンドを使います（condaコマンドでもできるらしいが見たことが無い）．

### 1.1_pipコマンドを使う理由

- condaコマンドよりも数多くのパッケージをインストールできる．
- 同じパッケージでも，condaコマンドよりもパッケージのバージョンが新しいことがある．
- condaコマンドよりもインストールする時間が短い．
- （パッケージの作成者にとって）パッケージの作成が簡単．

### 1.2_condaコマンドを使う理由

- パッケージのインストールだけでなく，仮想環境の取り扱いなども同じcondaコマンドを使ってできる．
- Miniforgeリポジトリを使って構築したPython環境では，パッケージをインストールするための標準コマンド．

### 1.3_2つのコマンドを使い分ける理由

- pipコマンドでインストールしたパッケージは，pipコマンドを使って更新や削除を行う必要があります．
- **Miniforge**リポジトリを使ってPython環境を構築しているので，condaコマンドを使ってパッケージをインストールするべきですが，condaコマンドでインストールできないパッケージもあります（visautilsパッケージがまさしくそう）．
- 両方のコマンドを併用してパッケージをインストールするのは望ましくないのですが，基本はcondaコマンドでパッケージをインストールしておき，condaコマンドでインストールできないパッケージのみ，pipコマンドを使ってインストールする，2つのコマンドを使い分けるのが良いのではないかと思います（あくまでも個人的見解ですが）．
- 安全を期すのであれば，仮想環境を使って，ベースはcondaコマンドのみ使ってパッケージをインストールしておき，pipコマンドを使う必要のあるパッケージは，新たに仮想環境を追加して，そちらにインストールするのも悪くないと思います（手間がかかりますが）．

### 1.4_パッケージのリポジトリを調べる方法

- condaコマンドを使って，インストールされたパッケージのリポジトリを調べることができます．
- 下記に示すように，conda listコマンドを実行してみて下さい．インストールされている全てのパッケージの一覧が表示されますが，4列目のChannel欄に，リポジトリが表示されます．
- conda-forgeと表示されるのはcondaコマンドでインストールされたパッケージ，pypiと表示されるのはpipコマンドでインストールされたパッケージです（visautilsパッケージのように，ローカルファイルからインストールされたパッケージも，pypiと表示されます）

```txt
conda list

# packages in enviromnet at c:\users\ユーザ名\miniforge3:
#
# Name      Version   Build            Channel
babel       2.18.0    pyhcf101f3_1     conda-forge
bacjrefs    1.3.0     py313h2a31948_0  conda-forge
...
```
