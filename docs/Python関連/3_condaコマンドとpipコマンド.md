## 1_condaコマンドとpipコマンドの違い

- Pythonは，数多くの有益パッケージを利用でき，それが最大の特長となっています．
- 但し，パッケージをインストールするのに，condaコマンドとpipコマンドという2つのコマンドがあります（他にもあります）．これらのコマンドの違いは何なんでしょうか．
- 最大の違いは，パッケージのリポジトリ（インストールするパッケージが収納先されている場所）が異なります．condaコマンドでインストールするパッケージはconda-forgeリポジトリ，pipコマンドでインストールするパッケージはPyPIリポジトリとなります．但し，visautilsパッケージのように，ローカルファイルからインストールするケースでは，pipコマンドを使います（condaコマンドでもできるらしいが見たことが無い）．

### 1.1_pipコマンドを使う理由

- condaコマンドよりも数多くのパッケージをインストールできる．
- 同じパッケージでも，condaコマンドよりもパッケージのバージョンが新しいことがある．
- condaコマンドよりもインストールする時間が短い．
- （パッケージの作成者にとって）パッケージの作成が簡単．

### 1.2_condaコマンドを使う理由

- パッケージのインストールだけでなく，仮想環境の取り扱いなどもできる．
- Miniforgeリポジトリを使って構築したPython環境では，パッケージをインストールするための標準コマンド．

### 1.3_2つのコマンドを使い分ける理由

- pipコマンドでインストールしたパッケージは，pipコマンドを使って更新や削除する必要があります．
- Miniforgeリポジトリを使ってPython環境を構築しているので，condaコマンドを使ってパッケージをインストールするべきですが，condaコマンドでインストールできないパッケージもあります（visautilsパッケージがまさしくそう）．
