## 1_Miniforgeリポジトリを使ったPython環境の構築

- WindowsマシンにPython環境を構築（インストール）する方法は幾つもあります．ここでは，**Miniforge**リポジトリを使ってPython環境を構築する手順を紹介します．
- Miniforgeリポジトリ以外の方法を使っても，Python環境を構築することはできます．Miniforgeリポジトリを使う方法が，他の方法に比べて優れているという訳ではありませんので，ユーザの好きな方法でPython環境を構築してもらっても全く問題ありません．
- 従来は，**Anaconda**リポジトリを使ったPython環境の構築が主流だったのですが，Anacondaリポジトリへアクセスが極端に集中したため，今では，一部のユーザ以外は，Anacondaリポジトリを使ってPython環境を構築するのは有償となっています．現時点（2026年3月現在）では，Miniforgeリポジトリを使ってPython環境を構築するのは無償です（将来的には分かりませんが）．
- **現時点（2026年3月現在）で，visautilsパッケージはPython3.13系のみ動作を保証しています**．これは，下記の理由によるものです．visautilsパッケージが3.14系以降のPythonで動作できるようになるには，Pythonnetパッケージ次第となります．
    - 最新のMiniforgeリポジトリを使ってPython環境を構築した際のPythonはPython3.13系であること
    - Pythonnetパッケージが，Python3.13系までしかサポートしていないため（Python3.14系以降はインストールに失敗する）

### 1.1_Miniforgeリポジトリを使ったPython環境の構築の手順

- [Miniforgeのダウンロード先](https://github.com/conda-forge/miniforge/)から，Windows版を選んでダウンロードします（画面右側のReleasesと書かれた文字をクリックし，（2026年3月現在最新の）**Release 26.1.1-3**を選んで，一番下側にある**Miniforge3-Windows-x86_64.exe**というファイルです）．
- 上記のサイトには，Release 26.1.1-3のMiniforgeリポジトリを使ってインストールされるPythonのバージョンは3.12と記述してありますが，実際にインストールされるPythonのバージョンは3.13でした．
- **Miniforge**をインストールします．  
    - 途中のダイアログで幾つか確認項目が表示されますが，全てデフォルトのままインストールします．
    - デフォルト以外の項目を選択しても良いですが，その時は，異なるPython環境となりますので，以下に説明は必ずしも当てはまりません．
    - デフォルト以外の項目を選択してインストールする場合は，以下の説明を適切な手順に置き換えて下さい．
- デフォルト以外の項目を選択してもPython環境の構築には問題ありませんが，それらの環境に対して全て説明するのは困難なので，デフォルトの項目を選択したケースに限定して説明を続けます．

| 確認項目                                    | デフォルト                                                     | 説明                                                                                                           |
| ------------------------------------------- | -------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| Install for:                                | Just Me (recommended)                                          | ユーザ毎にインストールする<br>Windows共有としてインストールする<br>の選択                                      |
| Create shortcuts (supported packages only). | チェックされている<br>それ以外の項目は全てチェックされていない | スタートメニューにインストールしたPython環境を開始するためのショートカット（**Miniforge Prompt**）が生成される |

- 以上の手続きで，WindowsマシンにPython環境を新規にインストールすることができました．

- **これ以降，Pythonを使った作業は全て，スタートメニューからMiniforge Promptを選択して起動したコマンドプロンプトで行って下さい**．
- このMiniforge Promptを使うことで，上記の手順でインストールしたPython環境が適用されるようになります．Windowsのコンソールを開いてPythonスクリプトを実行すると，WindowsPCにインストールされた別のPython環境が適用される可能性があります．必ず，このMiniforge Promptを使ってPythonスクリプトを実行して下さい．

### 1.2_condaコマンドを使ったパッケージのインストール

- Pythonには数多くの有益なパッケージがネット上に提供されており，それらのパッケージは**conda**コマンドを使ってインストールします．

#### 1.2.1_プロキシ対応

- condaコマンドを使ったパッケージのインストールは，外部へネットワークを接続します．外部へのネットワーク通信がプロキシを経由する場合は，Windowsでは**ユーザ環境変数**として，  

| 環境変数        | 値の設定例                                                 |
| --------------- | ---------------------------------------------------------- |
| **HTTP_PROXY**  | http://アカウント名:パスワード@プロキシアドレス:ポート番号 |
| **HTTPS_PROXY** | http://アカウント名:パスワード@プロキシアドレス:ポート番号 |

を設定して下さい．  
（上記の例はあくまでも参考例です．各組織毎にプロキシを設定する方法は異なりますので，適切な値を調べて設定して下さい）  

- Windowsのユーザ環境変数の設定は，「スタートボタン」→「設定」→「システム」→「バージョン情報」で，**デバイスの仕様**の関連リンクとして**システムの詳細設定**リンクが表示されるので，そちらを押下して下さい．
- **システムのプロパティ**ウィンドウが表示されるので，画面右下の**環境変数**ボタンを押下し，画面上段の，**ユーザ環境変数**設定欄に，**新規**ボタンを押下して，上記の環境変数を設定して下さい．

- 環境変数を設定した後は，設定した環境変数を反映させるため，スタートメニューから**Miniforge Prompt**を選択して，新たに起動して下さい（環境変数を設定した時に開いているMiniforge Promptでは，設定した環境変数は反映されません）

#### 1.2.2_最初に1回だけ行う手続き

- 下記に示すコマンドを実行して，condaコマンド本体と，インストールしたPython環境を最新に更新します．

```txt
conda update -n base conda
conda update --all --y
```
#### 1.2.3_必要なパッケージのインストール

- 下記のパッケージを，**conda**コマンドを使って，追加でインストールします．

```txt
conda install numpy scipy matplotlib pyvisa ruamel.yaml pythonnet
```
| パッケージ      | 用途                                    |
| --------------- | --------------------------------------- |
| **numpy**       | numpy配列やFFTなどの演算処理                    |
| **scipy**       | numpyパッケージで提供されていない演算処理                |
| **matplotlib**  | グラフを描画する                        |
| **pyvisa**      | 測定機器にSCPIコマンドを送受信する    |
| **ruamel.yaml** | YMALフォーマットのファイルを扱う                  |
| **pythonnet**   | .NET系言語からvisautilsパッケージを使う |

- **conda**コマンドを使って，パッケージをインストールするには，Miniforge Promptで，  
　　**conda install パッケージ名** 
と入力します．
- 具体的に，**numpy**パッケージをインストールするには，  
　　**conda install numpy**  
と入力します．
- 複数のパッケージを半角スペースで区切って，一度にインストールすることもできます．

### 1.3_VISA用ドライバのインストール

- Pythonで測定機器をコントロールするには，別途，**VISA**用ドライバ（**NI-VISA**など）をインストールする必要があります．所有している測定機器類に応じて，適したVISA用ドライバをインストールして下さい．

### 1.4_測定システムのvisautilsパッケージのインストール

- **visautils**パッケージ（Python製測定機器制御ドライバ）は，condaコマンドではなく，**pip**コマンドを使って，visautilsパッケージファイル（拡張子が.whlのファイル）からインストールします．
- バージョン**3.13**のPython用の**visautils**パッケージは，**visautils-x.y.z-cp313-cp313-win_amd64.whl**ファイルです（x.y.zはvisautilsパッケージのバージョンです）．下記のコマンドを実行して，visautilsパッケージをインストールして下さい．

```txt  
pip install visautils-x.y.z-cp313-cp313-win_amd64.whl
```
